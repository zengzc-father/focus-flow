# Focus Agent 架构设计

## 概述

Focus Agent 是一个**主动式AI Agent**，它不只是被动响应用户输入，而是持续感知用户环境、主动决策、执行任务，并从中学习改进。

## 1. 感知层 (Perception)

### 持续感知循环

```dart
每30秒执行一次:
  ├─ 获取当前使用统计
  ├─ 检测当前活动应用
  ├─ 读取设备状态（电量、网络、位置）
  ├─ 计算连续使用时长
  ├─ 检查目标完成情况
  └─ 构建环境快照(ContextSnapshot)
```

### 感知数据源

| 数据类型 | 获取方式 | 用途 |
|---------|---------|------|
| 应用使用时长 | UsageStatsManager | 判断使用模式 |
| 当前活动应用 | 轮询检测 | 识别高沉迷应用 |
| 屏幕状态 | 系统广播 | 统计亮屏时间 |
| 设备状态 | BatteryManager | 判断使用场景 |
| 时间/日期 | 系统时间 | 时间模式识别 |
| 历史决策 | 本地存储 | 避免重复提醒 |

## 2. 推理层 (Reasoning)

### 决策循环

```dart
每5分钟执行一次:
  ├─ 分析当前上下文
  ├─ 评估用户状态
  ├─ 判断是否需干预
  ├─ 调用LLM生成决策
  ├─ 解析决策JSON
  └─ 决定是否执行
```

### LLM提示词模板

```
【角色设定】
你是一位专业的学习助手AI Agent，帮助用户管理手机使用时间。

【当前环境】
- 时间: 20:30 (周三)
- 今日总使用: 4小时12分钟
- 当前应用: 抖音 (连续使用48分钟)
- 上次休息: 1小时前
- 今日目标: 屏幕时间<3小时 ❌

【决策选项】
1. do_nothing - 用户状态良好，不干预
2. gentle_reminder - 温和提醒建议休息
3. strong_intervention - 必须停止当前行为
4. suggest_activity - 推荐替代活动
5. create_rule - 创建自动化规则
6. adjust_goal - 动态调整今日目标

请分析并输出JSON格式决策...
```

### 决策输出格式

```json
{
  "reasoning": "用户已连续使用抖音48分钟，超过健康阈值，今日总使用也超标，建议干预",
  "decision": "suggest_activity",
  "confidence": 0.92,
  "action": {
    "type": "notification",
    "content": "已经刷了很久了，起来活动一下吧！",
    "timing": "immediate"

  "new_rule": {
    "name": "抖音超时提醒",
    "condition": {"app": "douyin", "consecutiveOver": 30},
    "action": {"type": "suggest_activity"}
  }
}
```

## 3. 执行层 (Execution)

### 执行动作类型

| 动作类型 | 描述 | 触发场景 |
|---------|------|---------|
| notification | 发送通知提醒 | 需要提醒用户时 |
| suggest_activity | 推荐替代活动 | 连续使用超时 |
| create_rule | 创建自动化规则 | 发现重复模式 |
| adjust_goal | 调整当日目标 | 目标设定不合理 |
| lock_app | 临时锁定应用 | 深夜过度使用 |

### 规则引擎

```dart
class SmartRule {
  // 触发条件
  final RuleCondition condition;
  
  // 执行动作
  final RuleAction action;
  
  // 评估函数
  bool isTriggered(ContextSnapshot context) {
    return condition.evaluate(context);
  }
}

// 示例规则
规则: "晚间使用限制"
  条件: 时间 > 20:00 AND 今日使用 > 240分钟
  动作: 发送通知 "今天使用已达标，建议休息"
```

## 4. 记忆层 (Memory)

### 记忆类型

| 记忆类型 | 存储内容 | 用途 |
|---------|---------|------|
| 用户画像 | 偏好时段、活动偏好、历史模式 | 个性化决策 |
| 决策历史 | 每次决策的上下文和结果 | 避免重复、效果评估 |
| 规则库 | Agent创建的自动化规则 | 持续自动化 |
| 目标记录 | 每日目标及完成情况 | 进度追踪 |

### 学习机制

```dart
// 决策效果反馈
void onDecisionExecuted(AgentDecision decision, UserReaction reaction) {
  // 用户对决策的反应：
  // - 立即采纳（好决策）
  // - 忽略（提醒力度不够）
  // - 关闭（时机不当）
  
  // 更新决策模型
  _learningModel.update(decision, reaction);
}
```

## LLM 集成方案

### 模型选择: Gemma 4E 2bit

- **大小**: ~2.6GB (适合移动端)
- **量化**: 4-bit，推理速度快
- **能力**: 足够处理决策生成、规则解析、对话理解

### 推理流程

```dart
// 1. 构建提示词
final prompt = buildPrompt(context, options);

// 2. 调用本地LLM
final response = await localLLM.generate(
  prompt,
  maxTokens: 512,
  temperature: 0.7,
);

// 3. 解析JSON决策
final decision = AgentDecision.fromJson(response);

// 4. 执行决策
if (decision.shouldAct) {
  await execute(decision.action);
}
```

### 性能优化

- **预加载**: App启动时加载模型到内存
- **量化推理**: 4-bit量化降低内存占用
- **上下文缓存**: 复用KV缓存加速连续推理
- **异步执行**: 推理在后台线程，不阻塞UI

## 工作流程示例

### 场景1: 首次创建规则

```
用户: "帮我设置规则，晚上8点后如果我用抖音超过30分钟就提醒我"

Agent:
  1. 解析意图 → 创建规则
  2. 提取参数:
     - 时间: 20:00之后
     - 应用: 抖音
     - 阈值: 30分钟
     - 动作: 通知提醒
  3. 生成规则JSON
  4. 保存到规则库
  5. 回复用户: "已创建规则'晚间抖音提醒'，将在条件触发时自动运行"

[后续自动执行]
当晚20:30，用户使用抖音32分钟 → 规则触发 → 发送通知
```

### 场景2: 主动干预

```
[Agent感知]
20:45: 检测到抖音已连续使用45分钟

Agent:
  1. 评估上下文:
     - 时间: 晚间
     - 使用时长: 超标
     - 上次提醒: 30分钟前
  2. LLM决策: "strong_intervention"
  3. 生成个性化消息:
     "已经刷了很久了，起来活动一下吧！"
  4. 推送通知 + 推荐眼保健操

用户反应: 点击"开始" → 记录成功干预
```

### 场景3: 智能规划

```
用户: "下午没课，给我规划一下"

Agent:
  1. 分析:
     - 时间: 14:00 下午
     - 今日使用: 2小时 (正常)
     - 历史: 下午常刷短视频
  2. LLM生成规划:
     "建议安排:
      14:00-14:30 复习上午课程
      14:30-14:35 休息
      14:35-15:00 完成作业
      之后可以适度休息"
  3. 创建临时提醒规则
  4. 在对应时间点发送提醒
```

## 关键设计决策

### 1. 为什么是Agent而非Chatbot?

| Chatbot | Agent |
|---------|-------|
| 被动等待用户输入 | 主动感知环境 |
| 单次对话 | 持续运行 |
| 无状态 | 有记忆和学习 |
| 仅回复 | 可执行动作 |

### 2. 为什么用本地LLM?

- **隐私**: 用户数据不上传
- **延迟**: 本地推理<500ms
- **离线**: 无网络也能工作
- **成本**: 无API调用费用

### 3. 规则 vs LLM决策?

- **规则**: 用于确定性逻辑（固定时间、固定阈值）
- **LLM**: 用于模糊决策（"是否该提醒"需综合判断）
