# Focus Flow Lite - 极简自律提醒

## 设计理念

> **简单、轻量、无感知**

- ❌ 不要Agent - 太复杂，后台耗电
- ❌ 不要LLM - 模型大，推理耗电
- ❌ 不要锁机 - 打扰使用
- ❌ 不要社交 - 增加复杂度
- ✅ 只要提醒 - 到点提醒，仅此而已
- ✅ 超低功耗 - 几乎不影响续航

---

## 核心功能（仅3个）

### 1. 连续使用提醒
```
设置: 连续使用超过 X 分钟提醒一次
默认: 45分钟

实现: 
- 不实时监听（耗电）
- 只在屏幕解锁时检查（无后台任务）
- 计算本次解锁到上次的时间差
```

### 2. 累计使用提醒
```
设置: 每天使用超过 X 小时提醒
默认: 3小时

实现:
- 每日首次解锁时清零
- 累加每次亮屏时长
- 达到阈值时提醒一次
```

### 3. 睡前提醒
```
设置: 几点提醒准备休息
默认: 22:30

实现:
- 系统闹钟一次提醒
- 不重复，只响一次
```

## 技术实现（极简）

### 不用的技术
- ❌ WorkManager周期性任务（耗电）
- ❌ 前台服务（常驻内存）
- ❌ UsageStatsManager（需要权限，数据量大）
- ❌ LLM推理（高耗电）
- ❌ 复杂数据库

### 用的技术
- ✅ 屏幕解锁广播（系统自带，无额外耗电）
- ✅ SharedPreferences（简单存储）
- ✅ 单次闹钟（系统服务）
- ✅ 通知栏（仅提醒时唤醒）

## 代码实现核心逻辑

### 使用计算（极简版）

```dart
// 不实时统计，只计算每次解锁到锁屏的时间
class SimpleUsageTracker {
  static DateTime? _lastUnlockTime;
  static int _todayTotalSeconds = 0;
  static int _currentSessionSeconds = 0;
  
  // 屏幕解锁时调用
  static void onScreenOn() {
    _lastUnlockTime = DateTime.now();
    
    // 检查是否跨天
    if (_isNewDay()) {
      _todayTotalSeconds = 0;
    }
  }
  
  // 屏幕锁定（或应用切换检测）时调用
  static void onScreenOff() {
    if (_lastUnlockTime == null) return;
    
    final duration = DateTime.now().difference(_lastUnlockTime!).inSeconds;
    _todayTotalSeconds += duration;
    _currentSessionSeconds += duration;
    
    // 检查是否触发提醒
    _checkThresholds();
    
    _lastUnlockTime = null;
  }
  
  static void _checkThresholds() {
    // 检查连续使用
    if (_currentSessionSeconds > 45 * 60) { // 45分钟
      NotificationService.show('已连续使用45分钟，休息一下吧');
      _currentSessionSeconds = 0; // 重置，避免重复提醒
    }
    
    // 检查累计使用（每天只提醒一次）
    if (_todayTotalSeconds > 3 * 3600 && !_alreadyRemindedToday) { // 3小时
      NotificationService.show('今日使用已超3小时，注意保护眼睛');
      _alreadyRemindedToday = true;
    }
  }
}
```

### 后台实现（零常驻）

```dart
// 不需要WorkManager持续运行
// 只需要监听系统广播

class ScreenStateReceiver extends BroadcastReceiver {
  @override
  void onReceive(Context context, Intent intent) {
    if (intent.action == Intent.ACTION_SCREEN_ON) {
      SimpleUsageTracker.onScreenOn();
    } else if (intent.action == Intent.ACTION_SCREEN_OFF) {
      SimpleUsageTracker.onScreenOff();
    }
  }
}

// AndroidManifest.xml 注册
// <receiver android:name=".ScreenStateReceiver">
//   <intent-filter>
//     <action android:name="android.intent.action.SCREEN_ON"/>
//     <action android:name="android.intent.action.SCREEN_OFF"/>
//   </intent-filter>
// </receiver>
```

## 功耗分析

### 原版Agent方案
```
每30秒感知循环:
- CPU唤醒: 30次/小时
- 内存占用: 50MB+
- 后台服务: 持续运行
- LLM推理: 偶尔高耗电
日均耗电: 5-10%
```

### Lite极简方案
```
仅屏幕状态变化:
- CPU唤醒: 用户解锁次数（约50次/天）
- 内存占用: 0（无后台服务）
- 后台服务: 无
- 推理计算: 无
日均耗电: <1%

几乎和系统自带功能一样省电
```
