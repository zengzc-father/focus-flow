# Focus Flow 混合架构设计

## 核心原则

```
┌─────────────────────────────────────────┐
│              用户使用场景               │
├─────────────────────────────────────────┤
│                                         │
│  场景A: 日常使用（90%时间）             │
│  ├─ 使用手机                          │
│  ├─ 锁屏/解锁                         │
│  ├─ 收到简单提醒（连续/累计/睡前）     │
│  └─ Agent不工作 ✓                     │
│                                         │
│  场景B: 打开App（10%时间）              │
│  ├─ 查看今日使用                      │
│  ├─ 和Agent对话（按需）                │
│  ├─ 生成规则/查看趋势                 │
│  └─ Agent激活工作 ✓                   │
│                                         │
└─────────────────────────────────────────┘
```

**关键洞察**: Agent的任务是"生成规则"和"分析总结"，不是"实时决策"

## Agent工作模式

### 启动时机
```
用户行为:
├─ 打开App首页 → 不加载Agent
├─ 点击"Agent"Tab → 加载Agent
├─ 点击"生成规则"按钮 → 加载Agent
└─ 点击"查看周报" → 加载Agent

Agent生命周期:
加载 (2-5秒) → 工作 → App切后台/关闭 → 自动释放
```

### Agent功能清单

| 功能 | 输入 | 输出 | 示例 |
|------|------|------|------|
| **规则生成** | 自然语言描述 | JSON规则 | "晚上8点刷抖音超30分钟提醒我" → `{条件,动作}` |
| **提醒文案** | 当前场景 | 个性化文案 | 连续45分钟 → "起来活动一下，喝杯热水吧" |
| **趋势总结** | 7天使用数据 | 周报文字 | "本周比上周少用1.2小时，周三周四是高峰期" |
| **规划建议** | 今日安排 | 时间建议 | "下午有2小时空闲，建议15:00-17:00学习" |

## 规则流转

```
用户输入自然语言
      │
      ▼
┌─────────────┐
│   Agent     │ ◄── 按需激活
│  理解意图   │
│  生成规则   │
└──────┬──────┘
       │ JSON规则
       ▼
┌─────────────┐
│  规则引擎   │ ◄── 常驻后台
│  保存规则   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  日常使用   │ ◄── 无Agent参与
│  规则触发   │
│  发送通知   │
└─────────────┘
```

**关键点**:
- Agent只负责"生成"规则
- 规则生成后，Agent可以休眠
- 规则执行由轻量级引擎处理
- 用户不打开App，Agent不消耗资源

## 功耗分析

### 后台常驻（Agent不激活时）
```
组件: 屏幕状态监听 + 规则引擎
内存: ~5MB
CPU: 仅在解锁/锁屏时唤醒（每天约50次）
耗电: <0.5%/天
```

### Agent激活时（用户使用App时）
```
组件: LLM模型 + Agent逻辑
内存: ~3GB（Gemma 4E 2bit）
CPU: 推理时占用
耗电: 仅用户使用App期间
```

### 对比
| 场景 | 原版Agent | 混合架构 |
|------|----------|---------|
| 日常待机 | 5-10%/天 | <0.5%/天 |
| 使用App | 额外3-5% | 3-5%（仅此时） |
| 用户体验 | 复杂 | 简洁 |

### AgentService 设计

```dart
class AgentService {
  bool _isLoaded = false;
  LocalLLM? _llm;
  
  // 按需加载
  Future<void> load() async {
    if (_isLoaded) return;
    _llm = await LocalLLM.load('gemma-4e-2bit.gguf');
    _isLoaded = true;
  }
  
  // 用完释放
  Future<void> unload() async {
    _llm?.dispose();
    _llm = null;
    _isLoaded = false;
  }
  
  // 功能1: 生成规则
  Future<SmartRule> generateRule(String naturalLanguage) async {
    final prompt = '生成规则: $naturalLanguage';
    final response = await _llm!.generate(prompt);
    return SmartRule.fromJson(response);
  }
  
  // 功能2: 生成提醒文案
  Future<String> generateReminderText(UsageContext context) async {
    final prompt = '场景: ${context.toString()}，生成提醒文案';
    return await _llm!.generate(prompt);
  }
  
  // 功能3: 生成周报
  Future<String> generateWeeklyReport(List<DailyUsage> data) async {
    final prompt = '数据: ${data.toString()}，生成周报';
    return await _llm!.generate(prompt);
  }
}
```

### App生命周期管理

```dart
class MyApp extends StatefulWidget {
  @override
  void didChangeAppLifecycleState(AppLifecycleState state) {
    switch (state) {
      case AppLifecycleState.resumed:
        // App回到前台，保持Agent加载（如果用户在看Agent页）
        break;
      case AppLifecycleState.paused:
      case AppLifecycleState.detached:
        // App切后台或关闭，释放Agent
        AgentService().unload();
        break;
    }
  }
}
```

## 用户使用流程

### 场景1: 纯监控（最常用）
```
用户: 正常使用手机
系统: 锁屏/解锁时记录时间
系统: 达到阈值时发送简单通知
Agent: 不参与 ✓
耗电: 极低 ✓
```

### 场景2: 创建规则（偶尔）
```
用户: 打开App → 点击Agent页
系统: 加载Agent (2-5秒)
用户: "帮我设置规则：晚上8点刷抖音超30分钟提醒我"
Agent: 理解 → 生成规则JSON → 保存
用户: 退出Agent页 或 App切后台
系统: 自动释放Agent
后续: 规则由常驻引擎执行
```

### 场景3: 查看周报（每周一次）
```
用户: 打开App → 点击周报
系统: 加载Agent
Agent: 分析7天数据 → 生成文字周报
用户: 阅读后退出
系统: 释放Agent
```

## 总结

| 问题 | 解决方案 |
|------|---------|
| Agent耗电 | Agent按需加载，用完释放 |
| Agent复杂 | Agent只负责生成配置，执行交给简单引擎 |
| 日常使用卡顿 | 日常无Agent，监控极简 |
| 个性化不足 | 打开App时用Agent生成个性化规则 |

**一句话**: 
> 平时像系统工具一样省电，打开时才变身智能助手。
