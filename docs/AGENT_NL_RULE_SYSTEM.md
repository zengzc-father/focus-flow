# Agent 自然语言规则生成系统

## 功能概述

Focus Flow 的 Agent 现在支持通过自然语言直接创建复杂的屏幕时间管理规则。用户只需用日常语言描述需求，Agent 即可自动解析并生成可执行的规则。

## 核心能力

### 1. 自然语言解析

支持多种自然语言表达方式：

```
时间表达:
- "晚上8点后"
- "下午3点"
- "深夜" / "凌晨"
- "上班时间"
- "从9点到12点"

时长表达:
- "30分钟"
- "1小时"
- "1.5小时"
- "半小时"
- "一刻钟"

周期表达:
- "周一到周五"
- "工作日"
- "周末"
- "每天"
- "周一、三、五"

场景表达:
- "上课期间"
- "自习时"
- "健身时间"
- "开会期间"

强度表达:
- "提醒我" (普通)
- "温和提醒" (静默)
- "强烈提醒" / "强制" (弹窗)
```

### 2. 支持的规则类型

#### 时间 + 应用 + 时长规则
```
用户: "晚上8点后刷抖音超过30分钟提醒我"
Agent解析:
  - 时间: 20:00-23:59
  - 应用: 抖音
  - 阈值: 30分钟
  - 动作: 发送提醒

生成的规则:
{
  "name": "晚上8点后抖音规则",
  "description": "20:00到23:59之间，使用抖音超过30分钟时提醒",
  "conditions": {
    "timeRange": "20:00-23:59",
    "consecutiveMinutes": 30
  },
  "action": {
    "type": "notify",
    "message": "抖音已使用30分钟，该休息一下了"
  }
}
```

#### 周期性规则
```
用户: "周一到周五晚上10点后还刷手机就强烈提醒"
Agent解析:
  - 周期: 周一、周二、周三、周四、周五
  - 时间: 22:00-23:59
  - 应用: 所有应用（手机）
  - 强度: 强烈
```

#### 场景化规则
```
用户: "上课期间玩游戏超过5分钟必须阻止我"
Agent解析:
  - 场景: 上课（基于日程）
  - 应用: 游戏
  - 阈值: 5分钟
  - 强度: 强制
```

#### 多条件复合规则
```
用户: "如果连续使用微信超过1小时，推荐我休息活动"
Agent解析:
  - 应用: 微信
  - 阈值: 60分钟
  - 动作: 推荐替代活动
```

### 3. Agent 回复示例

#### 成功创建规则
```
用户: "周一到周五晚上8点后刷抖音超过30分钟提醒我休息"

Agent回复:
✅ 已创建规则：周一到周五晚上8点后抖音规则

📋 规则详情：
   周一到周五，20:00到23:59之间，使用抖音超过30分钟时提醒

🔍 我识别到了：
   • 应用：抖音
   • 时长：30分钟
   • 时段：20:00-23:59
   • 周期：工作日

💡 规则已生效，我会在条件满足时提醒你。
```

#### 信息不足时的引导
```
用户: "提醒我别用太久"

Agent回复:
❓ 还需要以下信息：应用名称（如抖音、微信）、时长阈值（如30分钟、1小时）

💡 你可以这样说：
"周一到周五晚上8点后刷抖音超过30分钟提醒我休息"
"上课期间使用微信超过1小时强烈提醒"
```

## 技术实现

### 架构图

```
用户自然语言输入
       ↓
┌─────────────────┐
│ FocusAgentCore  │  ← 意图识别
│  _parseIntent() │
└────────┬────────┘
         ↓ createRule 意图
┌─────────────────┐
│ NLRuleParser    │  ← 自然语言解析
│     .parse()    │
└────────┬────────┘
         ↓ ParseResult
┌─────────────────┐
│ 提取的要素:      │
│ - 应用名称       │
│ - 时长阈值       │
│ - 时间范围       │
│ - 周期           │
│ - 场景           │
│ - 提醒强度       │
└────────┬────────┘
         ↓ SmartRule
┌─────────────────┐
│   RuleEngine    │  ← 规则存储与执行
│ addAgentRule()  │
└─────────────────┘
```

### 核心组件

#### 1. NLRuleParser（自然语言规则解析器）

```dart
// 解析入口
static ParseResult parse(String input)

// 解析结果
class ParseResult {
  final bool isValid;           // 是否解析成功
  final SmartRule? rule;        // 生成的规则
  final ParsedElements parsedElements;  // 解析出的元素
  final List<String> missingElements;   // 缺失的信息
}
```

#### 2. 意图识别增强

```dart
// 创建规则的多层级识别
if (lower.contains('如果') || lower.contains('超过') || ...) {
  return UserIntent(type: UserIntentType.createRule, ...);
}

// 场景化规则识别
if ((lower.contains('上课') || ...) &&
    (lower.contains('刷') || lower.contains('用') || ...)) {
  return UserIntent(type: UserIntentType.createRule, ...);
}
```

#### 3. 规则执行

```dart
// 规则被添加到 RuleEngine
final ruleEngine = RuleEngine();
await ruleEngine.addAgentRule(rule);

// RuleEngine 会:
// 1. 存储规则到 SharedPreferences
// 2. 监听 UsageTracker 的使用变化
// 3. 当条件满足时触发提醒
```

## 注意事项

1. **精确度**: 自然语言解析有局限性，复杂规则可能需要用户确认
2. **歧义处理**: "晚上"默认为 18:00-23:59，"深夜"为 00:00-06:00
3. **应用识别**: 支持 50+ 款中国热门应用，应用别名通过关键词匹配
4. **规则冲突**: 多个规则同时满足时，优先执行强度更高的提醒
