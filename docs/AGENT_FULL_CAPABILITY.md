# Focus Flow Agent 完整功能文档

## Agent 核心能力

Focus Flow 的 Agent 现在具备**完全代理用户需求**的能力，可以应对提醒自律方面的所有可能用户需求。

---

## 一、自然语言规则生成

### 核心能力
Agent 可以理解自然语言并自动生成监控规则，无需用户手动设置。

### 支持的表达方式

#### 1. 时间条件
```
"晚上8点后刷抖音..."
"下午3点开始..."
"深夜/凌晨..."
"从9点到12点..."
"上班/工作时间..."
```

#### 2. 时长阈值
```
"超过30分钟"
"达到1小时"
"用了20分钟"
"半小时"
"一刻钟"
```

#### 3. 周期设置
```
"周一到周五..."
"工作日..."
"周末..."
"每天..."
```

#### 4. 场景限制
```
"上课期间..."
"自习时..."
"健身时间..."
"开会期间..."
```

#### 5. 提醒强度
```
"提醒我..." (普通)
"温和提醒..." (静默)
"强烈提醒/强制..." (弹窗)
```

### 示例对话

**用户**: "周一到周五晚上8点后刷抖音超过30分钟提醒我休息"  
**Agent**: ✅ 已创建规则...
- 时间: 工作日 20:00-23:59
- 应用: 抖音
- 阈值: 30分钟
- 动作: 发送提醒

**用户**: "上课期间玩游戏超过5分钟必须阻止我"  
**Agent**: ⚠️ 已创建强烈提醒规则...
- 场景: 上课期间
- 应用: 游戏
- 阈值: 5分钟
- 强度: 强制弹窗

## 二、用户最高权限控制

### 核心理念
**用户永远有最高控制权**，Agent 只是协助执行用户意图。

### 功能列表

#### 1. 紧急跳过
当用户有正当理由需要继续使用时：
```
用户: "跳过这次提醒"
用户: "安静一会儿"
用户: "别管我，我在查资料"

Agent: ✅ 已跳过本次提醒
      ⏱️ 暂停干预: 15分钟
```

#### 2. 全局暂停
用户需要完全自由时间时：
```
用户: "今天全部暂停"
用户: "接下来3小时别管我"
用户: "周末让我自由使用"

Agent: 🌴 已进入自由模式
      ⏱️ 持续时间: 3小时
      这段时间我不会干预你，但数据仍在记录。
```

#### 3. 临时调课/改时间
突发情况时灵活调整：
```
用户: "今天高数课取消了"
用户: "英语课改到下午3点"
用户: "这周都不用上课"

Agent: ✅ 已取消高数课监控（4月18日）
```

#### 4. 临时自律模式
用户主动要求帮助自律：
```
用户: "接下来2小时别让我刷抖音，我要写论文"
用户: "帮我自律，现在到下午5点不能玩游戏"
用户: "我要专注学习，防止刷手机"

Agent: 💪 自律模式已启动！
      ⏱️ 持续时长: 2小时
      📱 监控应用: 抖音
      ⏰ 使用阈值: 1分钟
      💪 提醒强度: 强烈

      我会帮你盯着，加油！
```

## 三、完整意图识别矩阵

| 用户意图 | 关键词示例 | Agent 响应 |
|---------|-----------|-----------|
| **查询使用** | "今天用了多久"、"屏幕时间" | 今日使用统计 + 昨日对比 |
| **创建规则** | "如果...就..."、"超过...提醒" | 解析并创建规则 |
| **列出规则** | "有哪些规则"、"查看设置" | 规则列表展示 |
| **修改规则** | "暂停第2条"、"删除抖音规则" | 执行相应操作 |
| **询问建议** | "建议做什么"、"怎么休息" | 推荐休息活动 |
| **更新目标** | "目标改成3小时" | 更新每日目标 |
| **获取报告** | "给我周报"、"本周总结" | 生成使用报告 |
| **当前场景** | "现在有什么课" | 显示当前日程 |
| **紧急跳过** | "跳过"、"安静"、"别管" | 暂停15分钟 |
| **全局暂停** | "全部暂停"、"完全停止" | 按指定时长暂停 |
| **调课改期** | "调课"、"取消"、"改时间" | 调整日程监控 |
| **自律模式** | "别让我..."、"帮我自律" | 启动临时自律规则 |
| **通用对话** | 问候、帮助等 | 相应回复 |

## 四、技术架构

### 数据流

```
用户输入
   ↓
┌─────────────────┐
│ 意图识别        │ ← _parseIntent()
│ (14种意图类型)  │
└────────┬────────┘
         ↓
    ┌────┴────┬────────┬────────┬────────┐
    ↓         ↓        ↓        ↓        ↓
┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
│NLRule │ │User   │ │Tool   │ │通用   │ │建议   │
│Parser  │ │Override│ │Executor│ │对话   │ │系统   │
└───┬───┘ │Manager│ └───┬───┘ └───────┘ └───────┘
    ↓     └───┬───┘     ↓
┌───────┐    ↓    ┌───────────┐
│Smart  │    ↓    │ 数据查询  │
│Rule   │    ↓    │ 日程管理  │
└───┬───┘    ↓    └─────┬─────┘
    ↓        ↓          ↓
┌───────┐   ↓     ┌───────────┐
│Rule   │←──┴────→│ 用户反馈  │
│Engine │         │ (自然语言) │
└───────┘         └───────────┘
```

### 核心组件

#### 1. NLRuleParser (自然语言规则解析器)
- 提取应用名称
- 提取时间条件
- 提取时长阈值
- 提取周期设置
- 提取场景限制
- 提取提醒强度

#### 2. UserOverrideManager (用户覆盖管理器)
- 临时调课/改时间
- 全局暂停监控
- 紧急跳过提醒
- 临时自律模式
- 用户意图最高优先级

#### 3. FocusAgentCore (Agent核心)
- 意图识别
- 任务分发
- 结果生成
- 消息管理

## 七、总结

Focus Flow Agent 现在可以：

✅ **理解自然语言** - 用户用日常语言描述需求  
✅ **创建复杂规则** - 自动解析时间、应用、阈值等  
✅ **响应突发情况** - 调课、取消、暂停等  
✅ **支持主动自律** - 用户主动要求帮助时强力干预  
✅ **尊重用户主权** - 紧急跳过、全局暂停等最高权限  
✅ **全面覆盖需求** - 14种意图类型应对所有场景  

**Agent 不是用户的老板，而是用户的助手。**
**最终的控制权永远在用户手中。**
