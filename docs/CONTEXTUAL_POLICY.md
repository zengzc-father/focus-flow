# 场景感知策略文档

## 核心原则

> **人文精神：理解人类真实生活场景，不做机械的"监狱看守"**

```
❌ 错误做法：任何时间娱乐超过5分钟都警告
✅ 正确做法：
   - 上课时间：娱乐3分钟提醒（需要专注）
   - 健身时间：音乐随意听（合理需求）
   - 下课时间：完全不干预（自由支配）
```

## 策略矩阵

### 上课/自习时段 (EventType.course/study)

| 应用类型 | 阈值 | 说明 |
|---------|------|------|
| 娱乐(抖音/游戏) | 3分钟提醒 | 上课刷视频不合理 |
| 通讯(微信/QQ) | 10分钟提醒 | 回消息可以理解，但别聊太久 |
| 工具(相机/计算器) | 20分钟提醒 | 拍板书、计算是正当使用 |
| 学习(知乎/笔记) | 60分钟 | 鼓励使用，很少提醒 |
| 音乐 | 0(不推荐) | 上课不适合听歌 |

**示例消息：**
- 抖音3分钟："已经刷3分钟了，高数课还顺利吗？适度放松就好~"
- 微信12分钟："回消息用了12分钟，高数课别落下太多哦~"

### 健身时段 (EventType.exercise)

| 应用类型 | 阈值 | 说明 |
|---------|------|------|
| 娱乐 | 5分钟 | 组间休息可以刷一下 |
| 通讯 | 15分钟 | 休息时可以回消息 |
| 工具 | 10分钟 | 记录训练数据 |
| 音乐 | 120分钟 | 🎵 **鼓励使用！** 健身就该听音乐 |

**示例消息：**
- 抖音5分钟："组间休息刷抖音5分钟了，该下一组了~"
- 音乐："继续加油！音乐伴你运动💪" (不提醒，鼓励！)

### 会议时段 (EventType.meeting)

| 应用类型 | 阈值 | 说明 |
|---------|------|------|
| 娱乐 | 2分钟 | 会议中刷视频很不礼貌 |
| 通讯 | 20分钟 | 可能是工作消息 |
| 工具 | 30分钟 | 记录会议内容 |

### 自定义/休息时段 (EventType.custom)

| 应用类型 | 阈值 | 说明 |
|---------|------|------|
| 所有类型 | 30分钟 | **宽松模式，自由支配** |

> **重要：下课时间完全不干预！**

## 关键场景示例

### 场景1：高数课刷抖音

```
14:00 高数课开始
14:03 抖音 3分钟
      → 达到阈值
      → Agent: "已经刷3分钟了，高数课还顺利吗？适度放松就好~"

14:06 抖音继续 6分钟
      → 超过2倍阈值
      → Agent: "高数课时间已用6分钟刷抖音，有点久了，现在放下，等下课再玩吧！"

15:30 高数课结束
      → Agent: "高数课结束！这节课专注度85%，娱乐使用了8分钟。"
      → **下课自由时间，不再干预**
```

### 场景2：健身听音乐

```
16:00 健身开始
16:30 音乐播放 30分钟
      → **不提醒** (音乐阈值120分钟)
      → 用户享受运动，完全支持

17:00 健身结束
      → Agent: "健身完成！音乐伴你运动1小时，很棒💪"
```

### 场景3：上课时间统计独立

```
上午：
  08:00-09:30 高数课
    - 手机使用：5分钟（微信）
    - 专注度：96%
  
  09:30-10:00 课间休息
    - **不统计到上课** (独立时段)
    - 刷抖音20分钟：不干预

下午查询：
  用户: "今天高数课我专注吗？"
  Agent: "高数课专注度96%，只用了5分钟微信，很棒！"
  (课间刷的20分钟不计入高数课)
```

### 场景4：多时段独立监控

```
一天的活动：
├─ 08:00-09:30 英语课 (策略: 严格)
│  └─ 微信 5分钟 → 正常
│  └─ 抖音 4分钟 → 提醒
├─ 09:30-10:00 课间 (策略: 无)
│  └─ 刷抖音 → 不干预
├─ 10:00-11:30 专业课 (策略: 严格)
│  └─ 专注度 90%
├─ 14:00-15:30 健身 (策略: 音乐鼓励)
│  └─ 网易云 90分钟 → 👍 鼓励！
└─ 20:00-22:00 自习 (策略: 严格)

各时段独立统计，互不干扰
```

## 实现逻辑

```dart
class ContextAwareMonitor {
  // 每分钟检查
  void monitorCycle() {
    final context = getCurrentContext();
    
    // 1. 检查是否在日程中
    if (!context.isInScheduledTime) {
      // 下课/休息时间：完全不干预
      return;
    }
    
    // 2. 在活动中，根据活动类型+应用类型决定策略
    final policy = ContextualAppPolicy.getPolicy(event.type);
    final threshold = policy[appIntent]; // 获取该类应用的阈值
    
    // 3. 仅监控当前时段，不影响其他时段统计
    if (usageMinutes >= threshold) {
      sendIntervention(event, app, threshold);
    }
  }
  
  // 时段结束时下课
  void endTimeSlot(event) {
    saveSlotStats(); // 保存该时段统计
    clearCurrentSlot(); // 清零，下时段重新开始
  }
}
```

## 用户自定义

用户可以用自然语言调整：

```
用户: "我健身时可以多刷会儿抖音"
Agent: "已将健身时的娱乐阈值调整为15分钟"

用户: "我自习时完全不想碰手机"
Agent: "开启严格模式，自习时任何使用都会提醒"

用户: "我上课时可以偶尔回消息"
Agent: "已将上课时间通讯阈值调整为20分钟"
```

## 人文设计要点

1. **理解场景**
   - 健身听音乐 = 正常需求，不应该是"违规"
   - 上课拍板书 = 正当使用，不应被限制

2. **时段独立**
   - 课间刷视频 ≠ 上课不专注
   - 各时段独立统计，不相互拖累

3. **下课自由**
   - 非活动时间完全不干预
   - 给学生自由支配的权利

4. **渐进提醒**
   - 不是"立刻打断"
   - 3分钟温和提醒 → 6分钟强烈警告
   - 给用户自我纠正的空间
