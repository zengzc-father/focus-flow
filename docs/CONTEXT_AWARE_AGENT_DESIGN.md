# Focus Flow - 上下文感知Agent架构设计

## 核心升级点

从"通用监控"升级为"场景感知智能干预"：

```
之前: Agent知道"你今天用了4小时"
现在: Agent知道"你正在上高数课，但抖音用了20分钟，可能是走神了"
```

## 数据模型

### 1. 日程/课表

```dart
class ScheduleEvent {
  final String id;
  final String name;           // "高等数学"
  final EventType type;        // 课程 / 自定义活动
  final TimeSlot timeSlot;     // 时间
  final List<int> weekdays;    // [1,3,5] 周一三五
  final String? location;      // "教学楼A-301"
  final String? description;   // 额外说明
  final DeviceUsagePolicy policy; // 手机使用策略
}

enum EventType {
  course,      // 正式课程
  study,       // 自习时间
  exercise,    // 健身/运动
  meeting,     // 会议
  custom,      // 自定义
}

class DeviceUsagePolicy {
  final bool allowToolUsage;      // 允许工具使用（消息、相机等）
  final int entertainmentLimit;   // 娱乐使用限制（分钟）
  final bool requireCheckIn;      // 是否需要专注打卡
}
```

### 2. 时段使用分析

```dart
class TimeSlotUsage {
  final String scheduleEventId;   // 关联的日程
  final DateTime date;            // 日期
  final TimeSlot slot;            // 时段
  
  final int totalUsageSeconds;    // 总使用时长
  final int toolUsageSeconds;     // 工具使用时长
  final int entertainmentSeconds; // 娱乐使用时长
  
  final List<AppSession> sessions; // 详细使用记录
  final double focusScore;        // 专注度评分 0-100
}
```

### 3. 应用分类（意图识别基础）

```dart
enum AppIntent {
  tool,          // 工具性使用
  communication, // 通讯
  entertainment, // 娱乐
  study,         // 学习
  unknown,       // 未知
}

// 应用意图映射表
final appIntentMap = {
  // 通讯工具 - 上课/健身时短暂使用可接受

  'com.alibaba.android.rimet': AppIntent.communication, // 钉钉

  // 学习工具 - 鼓励使用

};
```

## 典型场景

### 场景1: 上课走神检测

```
10:00 用户有高数课
10:05 系统检测到抖音使用
10:10 累计娱乐使用5分钟

Agent决策:
  当前场景: 高数课 (应专注)
  娱乐使用: 5分钟 (超过阈值)
  判断: 走神/不专注

行动:
  发送通知: "高数课上刷抖音5分钟了，跟得上老师讲的内容吗？
           现在放下手机，下课再放松~"
  界面显示: Agent卡片提示，提供"我知道了"按钮
```

### 场景2: 健身时间合理使用

```
16:00 用户规划健身时间
16:05 检测到微信使用2分钟
16:06 停止使用

Agent决策:
  当前场景: 健身时间
  使用类型: 通讯工具 (2分钟)
  判断: 查看消息，合理短暂使用

行动: 不干预
```

### 场景3: 学习时段分析

```
用户问: "今天下午自习专注吗？"

Agent:
  1. 查询14:00-17:00自习时段
  2. 分析该时段使用:
     - 总手机使用: 45分钟/180分钟
     - 娱乐使用: 30分钟 (抖音20分+微博10分)
     - 工具使用: 15分钟 (微信+笔记)
  3. 专注度评分: 75%
  
回复: "今天下午自习了3小时，专注度75%~
      手机用了45分钟，其中刷抖音20分钟有点多哦。
      不过比昨天进步了，昨天同一时段用了1小时手机！"
```

### 场景4: 自然语言添加课表

```
用户: "我周一上午8点到9点半有高数课，在A301教室"

Agent解析:
  - 事件: 高等数学
  - 时间: 周一 8:00-9:30
  - 地点: A301教室
  - 类型: 课程
  - 默认策略: 专注模式（娱乐限制5分钟）

回复: "已添加高数课，每周一上午8:00-9:30，A301教室。
      上课期间我会帮你监控手机使用，超过5分钟娱乐使用会提醒你~"
```

## 实现优先级

1. **P0 - 日程数据层**
   - ScheduleEvent模型
   - ScheduleRepository存储
   - 自然语言解析（提取时间、地点、事件）

2. **P0 - 时段使用分析**
   - TimeSlotAnalyzer
   - 应用意图分类器
   - 专注度计算

3. **P1 - 增强Agent决策**
   - 场景识别（当前在做什么）
   - 策略执行（根据场景不同策略）
   - 个性化消息生成

4. **P2 - 自然语言交互**
   - 添加日程对话
   - 查询专注度对话
   - 策略设置对话
